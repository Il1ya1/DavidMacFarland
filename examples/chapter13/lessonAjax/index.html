Ajax

Введение

JavaScript - это мощное но не всесильное средство. Если вы ходите предсавить информацию из базы данных, отправить форму с заполнеными данными, загрузить новый html на страницу, поставить оценку в рейтинге и тд. вам потребуется обратиться к серверу, Ajax позволяет сделать это без перезагрузки сервера.

Что такое Ajax

Ajax - это аббревиатура от Asynchronous JavaScript и XML (Асинхронный JavaScript и XML)

Ajax не является отдельной технологией как JavaScript или CSS это набор технологий - языка JavaScript, браузера и веб себсера для получения и отображения обновленной информации без загрузки новой страницы.

Коротко говоря современные браузеры позволябт благодоря коду JavaScript посылать запрос на сервер, который присылает ответ браузеру некоторые данные JavaScript берез эти сведения и что нибудь с ними делает.

Чать того что позволяет делать Ajax :

  • Отоброжение нового HTML контента без загрузки страницы например кнопка "показать еще"
  • Отправление формы и получение неедленного ответа об успешной или не успешной отправки
  • Авторизация без ухода со страницы
  • Назначение рейтинга

Технология Ajax: Основый

Ajax не расматривается как самостоятельная технология это смесь ряда технологий, функционируюющих совместно для повышение работы пользователя. Ajax сводит вместе 3 компонента

  • Браузер - нужен для просмотра веб страницы и применения кода JavaScript. В современных браузерах есть встроенный обьект XMLhttpRequest, брагодоря ему код JavaScript может общаться с веб сервером и получать в ответ информацию.
  • JavaScript выполняет главные функции ajax: посылает запрос на сервер, ожидает ответ, обрабатывает его и (как правило) обновляет страницу добвляя новые данные или изменяя облик страницы.

    Получив информации код JavaScript обновляет страницу - представляет новые записи базы данных или например сообщает посетителю что он успешно авторизован. Обновление веь страницы включает маникулирование ее обьектной маделью (DOM) добавление, удаление, изменение

  • Веб-сервер получает запросы от браузера отправляет ему информацию. Сервер может вернуть html код, документ XML или данные JSON. Например при получении данных из формы он может добавить ее базу данных и отослать обратно сообщение подтверждающие что данные добавлены.

Главный смысл Ajax заключается в том чтобы позволить коду JavaScript общаться с сервером и получать от него информацию (данные). В большинстве случаев это подразумевает наличие дополнительного сценария, выполняемого на веб сервере и рушающего задачи, которые не под силу JavaScript.

Обработка данных с сервера

Браузер посылает запрос на сервер с помощью объекта XML HttpRequest и ожидает ответ. Когда он приходит, функция обратного вызова обрабатывает его. Ей передаются несколько аргументов, первый из которых ( самый важный) и определяет данные, возвращаемые сервером

Вы можете форматировать сведения, возвращаемые сервером, любыми способами. Серверный сценарий возвращает число, слово, абзац текста или целую страницу. В случаях, когда сервер присылает большой объем информации (подобно группе записей базы данных), он часто использует форматы XML или JSON

Второй аргумент функции обратного вызова представляет собой строку, указывающую статус ответа. Чаще всего статус success означает, что сервер успешно обработал запрос и возвратил данные. Однако иногда обработка запроса не удается, например, был запрошен файл, который уже не существует, или произошла ошибка в серверной программе. В таких случаях функция обратного вызова получает сообщение со статусом error.

Функция обратного вызова определенным образом обрабатывает данные и, как правило, обновляет веб-страницу, заменяя отосланную форму результатами, полученными с сервера, или, например, просто отображая на странице сообщение «request successful�. Процесс обновления содержимого страницы облегчается функциями j Query h trnl ( ) и text (), описанными в разделе «Добавление содержимого на страницу
